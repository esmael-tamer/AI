You are an expert full-stack AI agent. Build a complete MVP platform (Web + Mobile) for “Media Trend”.
Do NOT focus on colors, UI styling, or design references. Prioritize functionality, structure, roles, flows, and data.

========================================================
PRODUCT SUMMARY
========================================================
Build a “Demo-first” platform:
- The user types OR speaks their store idea.
- An AI assistant asks only the minimum missing questions (max 6–8).
- The platform instantly generates a fully interactive DEMO ecommerce store (no signup required) and shows it.
- Only after the demo is visible, the user logs in to continue and submits a request that reaches Media Trend.
- Media Trend team follows up and completes real activation (payments, shipping, warehousing).

Business model (non-negotiable):
- Unlimited products (no limits).
- No monthly/yearly subscriptions.
- Commission on sales only.
- Checkout remains DEMO until required integrations are activated.

Primary markets:
- Kuwait & Saudi (global-ready).
Languages:
- English first, but keep i18n-ready for Arabic (RTL) later.

========================================================
REQUIRED PORTALS (SEPARATE ENTRY)
========================================================
1) Public website (marketing site)
2) Customer Portal (web + mobile) for user dashboard
3) Admin Portal (web only) for internal team

Hard rules:
- Customers must never access admin routes.

- Customer login must be separate from admin login.

========================================================
CORE USER FLOW (MANDATORY)
========================================================
A) DEMO FIRST (no signup upfront)
1) User lands on Home.
2) User enters store idea by:
   - typing (text)
   - or speaking (voice)
3) AI asks up to 6–8 short questions (only what’s missing).
4) System generates StoreConfig JSON and creates a DEMO store record (guest):
   - store.ownerId = null
   - store.sessionId = guestSessionId
   - store.status = DEMO
5) Redirect user directly to the demo store URL:
   /s/[storeSlug]?demo=1
6) Demo store is fully interactive:
   - Home, Shop, Categories, Product detail, Cart, Checkout (demo), Policies, Order tracking (demo)

B) LOGIN ONLY AFTER DEMO IS VISIBLE
7) Demo store page must show a prominent CTA:
   “Continue / Activate”
8) Clicking CTA redirects to Customer Portal login/signup:
   /portal/login or /portal/signup
9) After login:
   - attach store to user:
     store.ownerId = user.id
     store.sessionId = null
   - redirect to customer dashboard store page:
     /portal/stores/[storeId]

C) SUBMIT REQUEST TO MEDIA TREND (MANDATORY)
10) In /portal/stores/[storeId], show a stepper “Complete Request”:
   Step 1: confirm store basics (name, country, contact)
   Step 2: select what to activate:
     - payments activation
     - shipping activation
     - warehousing activation (optional)
   Step 3: collect required info:
     - contact name
     - phone/whatsapp
     - email (optional)
     - business details (brief)
     - pickup address (for shipping)
     - notes
   Step 4: submit request

11) On submit:
   - create Lead record containing all inputs + demo store link
   - create Ticket records for each selected activation
   - set ticket status = Pending
   - set store status = Pending_Activation
   - show confirmation “Request received; our team will contact you”
   - provide a WhatsApp click-to-chat button (configurable number)

D) ACTIVATION RULE
12) Checkout must remain DEMO until:
   paymentsStatus = live AND shippingStatus = live
   (warehousing optional)

========================================================
ADDITIONAL SERVICES (SEPARATE, NOT AUTO-BUILD)
========================================================
Add two standalone services that generate an AI brief and then submit a request (requires login):
1) “Ad Campaign Launch” (all platforms)
2) “Social Account Management”

Service behavior:
- Public service page explains the service + CTA “Try AI Planner”
- AI Planner asks a few questions and outputs a structured brief (JSON):
  - For ads: goal, budget range, platforms list, country, industry, timeframe
  - For management: platforms, posting frequency, goals, brand info, market
- When user clicks “Submit Request”:
  - requires /portal/login if not logged in
  - creates Lead + Ticket:
    - ads_launch_request OR account_management_request
    - status = Pending
- Admin sees these requests in the same pipeline.
